# 🎮 Две интерактивные игры про математику отношений

Дружелюбные интерактивные объяснения в духе игры Nicky Case
[*The Evolution of Trust*](https://ncase.me/trust/). Обе игры полностью статичны:
чистый HTML/CSS/JS, без фреймворков, сборщиков, CDN и сети.

- 👰 **Привередливая невеста** (`index.html`) — задача о секретаре / правило 37%.
- 💘 **Игра в сваху** (`stable-marriage.html`) — задача о стабильных браках,
  алгоритм Гейла–Шепли. Все персонажи рисуются процедурно через SVG:
  собери пары мышкой, устрой хаос, запусти алгоритм по шагам и автоматически,
  проверь результат на прочность и открой сюрприз в песочнице
  («кто предлагает — тому и выгоднее»).

Игры связаны ссылками в сайдбаре, так что можно переключаться между ними.

# 👰 Привередливая невеста

Маленькое дружелюбное интерактивное объяснение классической **задачи о секретаре**
(она же «задача о привередливой невесте» / задача оптимальной остановки), сделанное
в духе игры Nicky Case [*The Evolution of Trust*](https://ncase.me/trust/).

Ты играешь за невесту, которая выбирает жениха из очереди. Каждый жених появляется
один раз — его нужно **сразу выбрать** или **отпустить навсегда**. Цель — заполучить
самого лучшего жениха во всей очереди. А потом игра показывает удивительно красивую
оптимальную стратегию: **смотри ~37%, потом выбирай.**

Без фреймворков. Без сборщиков. Без сервера. Без внешних ассетов. Три файла, которые
можно просто открыть в браузере.

## ✨ Что внутри

Игра — короткая интерактивная история с постоянным меню сцен слева:

1. **Вступление** — правила и милые нарисованные (SVG) персонажи.
2. **Первая попытка** — встречаешь 12 женихов по одному и пытаешься выбрать лучшего
   (видны только *относительные* подсказки, скрытый рейтинг — никогда).
3. **Что происходит?** — раскрытие всей очереди и мягкое объяснение, почему выбирать сложно.
4. **Стратегия 37%** — ползунок + кнопка «Запустить 1000 симуляций» + живой график
   на canvas «успех vs процент наблюдения». Пик — около 37%.
5. **Почему 37%?** — объяснение простыми словами (магическая доля — `1/e ≈ 36.8%`)
   с линией «фаза наблюдения / фаза выбора».
6. **Песочница** — крути параметры: число женихов, процент наблюдения, число симуляций
   и правило «взять последнего, если никто не побил рекорд». Видно: шанс выбрать лучшего,
   средний рейтинг выбранного, сколько раз невеста осталась одна, и эмпирически лучший
   процент наблюдения.

### Точное правило стратегии

1. Пропустить первые `k = floor(n * threshold)` женихов — просто наблюдать.
2. Запомнить лучший рейтинг среди пропущенных.
3. Выбрать **первого** следующего, кто побил этот рекорд.
4. Если никто не побил: по умолчанию невеста остаётся **одна** (это проигрыш —
   классическая трактовка). В песочнице можно включить «взять последнего».

## 🚀 Запуск локально

**Вариант А — просто открыть.** Дважды кликни по `index.html`. Всё работает прямо из
файловой системы, без сервера и без сети.

## 🌐 Публикация на GitHub Pages

1. Создай репозиторий и запушь эти файлы в ветку `main`:

```bash
git init
git add .
git commit -m "Привередливая невеста"
git branch -M main
git remote add origin https://github.com/<you>/<repo>.git
git push -u origin main
```

2. На GitHub: **Settings → Pages → Build and deployment**.
3. **Source** — *Deploy from a branch*, выбери ветку **`main`** и папку **`/root`**, **Save**.
4. Через минуту игра будет доступна по адресу `https://<you>.github.io/<repo>/`.

Всё статично, пути относительные — дополнительная настройка не нужна.

## 💘 Игра в сваху (Stable Marriage / Gale–Shapley)

Отдельная игра в `stable-marriage.html`. О чём она:

- **Задача о стабильных браках:** как разбить людей на пары так, чтобы не нашлось
  двоих, которые предпочли бы друг друга своим партнёрам (и сбежали бы вместе).
- **Алгоритм Гейла–Шепли (1962):** свободный кандидат делает предложение лучшему
  из ещё не спрошенных; принимающая сторона держит лучшего ухажёра, остальным —
  отказ; отвергнутый снова свободен. Повторять, пока свободные не кончатся.
- **Сюрприз:** результат зависит от того, кто делает предложения — предлагающая
  сторона всегда получает самый выгодный для себя стабильный вариант.
- **Персонажи** генерируются процедурно из случайных деталей (причёски, усы, очки,
  монокль, цилиндры, бабочки и т.д.) — только SVG, никаких картинок.
- **Где применяется:** распределение выпускников медвузов по больницам, учеников по
  школам, обмен донорскими почками. Нобелевская премия по экономике 2012 (Шепли, Рот).

## 🧠 Немного математики

Для `n` женихов: если наблюдать долю `r`, а потом брать следующего «рекордсмена»,
вероятность выбрать самого лучшего максимальна при `n → ∞` в точке `r = 1/e ≈ 0.368`,
и сама эта вероятность тоже стремится к `1/e ≈ 36.8%`. Попробуй побить это в песочнице —
в среднем не получится. 😉
